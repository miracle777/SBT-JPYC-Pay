# 🚀 SBT JPYC Pay - 利用ガイド

> **⚠️ 重要な前提条件**  
> SBT発行にはインターネット接続が必要です。企業・店舗利用には専用サーバー設定が必須です。

## 📱 このアプリについて

**SBT JPYC Pay** は、SBT（Soulbound Token）技術を活用したスタンプカード発行・管理システムです。

### ✅ できること（オフライン対応）
- SBTテンプレートの作成・編集
- ローカルデータの管理・検索
- QRコード決済画面の表示
- PWAアプリとしての動作

### ❌ インターネット接続が必要な操作
- **SBTの実際の発行** - Pinata (IPFS) への画像・メタデータアップロード
- **ブロックチェーン記録** - Polygon等ネットワークへのトランザクション送信
- **新規画像のIPFS保存** - 分散ストレージへのアップロード

## 🔒 データ管理・プライバシー

### 📍 データ保存場所
- **ローカルストレージ** - あなたのデバイスのIndexedDB/localStorage
- **サーバーなし** - 開発者のサーバーにデータは保存されません
- **ブロックチェーン** - SBT発行データは分散台帳に公開記録
- **IPFS** - SBTメタデータ・画像は分散保存

### 🛡️ プライバシー保護
- 個人情報の収集なし
- ウォレット秘密鍵は取得・保存しません
- 取引履歴は外部送信されません
- 完全なローカル管理

### 💾 データのバックアップ
- **定期的なエクスポート推奨** - 設定画面からJSON形式で出力可能
- **アプリ削除時にデータも削除** - 事前のバックアップが重要
- **複数デバイス間でのデータ共有** - エクスポート/インポートで実現

## 🔧 技術仕様

### 対応環境
- **ブラウザ** - Chrome, Firefox, Safari, Edge（最新版）
- **OS** - Windows, macOS, Android, iOS
- **PWA対応** - ホーム画面追加でアプリ化可能

### 必要なサービス
- **MetaMask** - ウォレット接続
- **Polygon Network** - メインチェーン（商用利用時）
- **Pinata** - IPFS分散ストレージ

### 技術スタック
- Frontend: React + TypeScript + Vite
- Blockchain: ethers.js v6, Solidity
- Storage: Pinata (IPFS)
- Styling: Tailwind CSS
- PWA: Service Worker, Manifest
